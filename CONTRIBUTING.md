# Contributing

This marketplace is **curated**, not open-PR.

## What that means

We don't accept third-party plugin submissions here. Every plugin in this marketplace is something we use in our own work and have committed to maintaining publicly. New plugins get added when we have something stable enough that we'd be embarrassed *not* to share it.

If you've forked a plugin and built something better, that's great — please publish it under your own marketplace. Claude Code handles multiple marketplaces side-by-side without any conflict.

## What we welcome

- **Issues**: bug reports, unexpected behavior, suggestions, questions about how a plugin works
- **Pull requests on existing plugins**: small bugfixes, doc corrections, typo fixes, additional examples — these are fair game and we'll review them
- **Discussions**: design ideas, feature requests, conversations about how a plugin could grow

## What gets declined

- **New plugins** — we maintain a small surface intentionally; we'd rather have five plugins we know cold than fifty we vaguely steward
- **Refactors of internals without a clear user-facing benefit**
- **Renames or breaking changes** without prior discussion in an issue

## Reporting a security issue

If you find something that looks like a leaked credential, an exfiltration path, or anything else that shouldn't be in a public repo, please email `brennan.gdwn.consulting@gmail.com` directly rather than opening a public issue.
